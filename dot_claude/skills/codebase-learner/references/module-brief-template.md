### 设计意图注释要点

为每个代码片段列出右侧需要解释的要点：
- {这个设计解决了什么问题}
- {为什么选这种方式而不是替代方案}
- {这种设计的 trade-off}
- {在什么条件下这个设计会失效}

### 交互元素

选择此单元需要的元素：

- [ ] **代码 ↔ 设计意图** — 使用哪些代码片段
- [ ] **测验** — {数量} 道题，类型：{架构判断 / 设计决策 / 代码追踪 / 方案对比}。简述每题的角度
- [ ] **数据流动画** — 角色：{列表}。步骤：{高亮和 packet 移动序列}
- [ ] **架构图** — 包含哪些区域和组件
- [ ] **调用链追踪** — 入口点：{文件:方法}，经过哪些层，终点
- [ ] **对比视图** — 对比什么：{A vs B}，结论：{为什么选 A}
- [ ] **模式卡片** — 展示哪些设计模式或工程实践
- [ ] **文件树** — 展示哪部分目录结构
- [ ] **步骤卡片** — 描述什么流程
