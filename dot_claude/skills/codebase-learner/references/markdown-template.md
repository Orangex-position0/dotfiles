# Markdown 输出模板

> **When to read this:** Phase 3 组装 Markdown 输出时。定义学习笔记的结构规范。

## 输出结构

Markdown 输出为一个目录，包含一个主文件和可选的子文件：

```
project-name-notes/
  README.md              ← 项目快照 + 学习路径索引
  01-domain-model.md     ← 各学习单元（按主题命名）
  02-architecture.md
  03-data-access.md
  ...
  glossary.md            ← 项目术语表（如果有）
```

## 写作规范

### 代码块

- 必须标注语言（````java`、````typescript` 等）以启用语法高亮
- 必须标注文件路径和行号范围
- 选择 5-15 行能体现设计模式的关键片段
- 代码后跟"设计意图"说明

### 章节层级

- `#`：单元标题（每文件一个）
- `##`：主题（3-5 个/单元）
- `###`：子主题（What / Why / How / Trade-offs）

### 链接

- 单元间用相对路径链接
- 引用外部资源用完整 URL
- 文件路径用反引号包裹

### 格式偏好

- 表格用于对比和总结
- 引用块（`>`）用于重要提示和设计要点
- 有序/无序列表用于步骤和要点
- 加粗用于关键术语首次出现
