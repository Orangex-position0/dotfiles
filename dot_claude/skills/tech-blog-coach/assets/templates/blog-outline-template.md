# 技术博客大纲模板（基于5部分固定结构）

## 第1步：确定主题

### 主题来源
从日常开发中选择：
- [ ] 刚解决的 Bug
- [ ] 刚学会的框架/设计模式
- [ ] 性能优化
- [ ] 踩坑经历

### 一句话总结
```
使用 [技术方案] 处理 [问题]
```

**你的主题**：使用 \_\_\_\_\_\_ 处理 \_\_\_\_\_\_

---

## 第2步：确定读者对象

### 写作目的
- [ ] 提供一套方法论
- [ ] 技术场景的解决方案
- [ ] 分享踩坑经验
- [ ] 其他：\_\_\_\_\_\_

### 目标读者
**技术水平**：比自己稍弱一级的开发者

**读者画像**：
- 技术背景：[如：后端开发者、前端工程师]
- 经验水平：[初级/中级/高级]
- 前置知识：[读者应该具备的基础]

---

## 第3步：结构设计（5部分固定模板）

### 1. 背景 & 问题（10-15%）

**目标**：让读者了解文章定位，明确能学到什么

**要点**：
- [ ] 介绍背景和问题场景
- [ ] 说明为什么这个问题重要
- [ ] 明确读者能学到什么
- [ ] 可选：设置悬念引发兴趣

**开头方式**：
- 从一个具体场景开始
- 从一个常见问题开始
- 从一次踩坑经历开始

---

### 2. 方案或原理（25-30%）

**目标**：讲解核心概念和技术方案

**要点**：
- [ ] 列出核心概念（分点讲述）
- [ ] 解释技术方案的原理
- [ ] 使用代码/图/列表辅助说明

**核心概念清单**：
1. 概念1：[说明]
2. 概念2：[说明]
3. 概念3：[说明]

**需要准备的视觉元素**：
- [ ] 架构图/流程图
- [ ] 代码框架示例
- [ ] 对比表格

---

### 3. 实现步骤（30-35%）

**目标**：问题 + 方案的具体实现

**步骤规划**：

#### 步骤 1：[步骤标题]
- 目标：[本步骤要达成什么]
- 代码：[是否需要代码示例]
- 注意：[关键说明]

#### 步骤 2：[步骤标题]
- 目标：[本步骤要达成什么]
- 代码：[是否需要代码示例]
- 注意：[关键说明]

#### 步骤 3：[步骤标题]
- 目标：[本步骤要达成什么]
- 代码：[是否需要代码示例]
- 注意：[关键说明]

---

### 4. 示例 / 踩坑（15-20%）

**目标**：完整示例 + 常见错误

**完整示例**：
- [ ] 提供完整的可运行代码
- [ ] 展示运行结果

**常见错误清单**：
1. 错误1：
   - 现象：[描述]
   - 原因：[分析]
   - 解决：[方案]

2. 错误2：
   - 现象：[描述]
   - 原因：[分析]
   - 解决：[方案]

**边界情况**：
- [ ] 情况1：[说明]
- [ ] 情况2：[说明]

**调试技巧**：
- [ ] 技巧1：[说明]
- [ ] 技巧2：[说明]

---

### 5. 总结（5-10%）

**目标**：总结 + 延伸 + 鼓励

**要点**：
- [ ] 快速总结3-5个核心要点
- [ ] 明确适用场景
- [ ] 诚实说明局限性
- [ ] 提供延伸方向
- [ ] 给予读者鼓励

**核心要点**：
1. [要点1]
2. [要点2]
3. [要点3]

**适用场景**：[说明]

**局限性**：[说明]

**延伸方向**：[说明]

---

## 第4步：编写初稿

**重要原则**：不要追求完美初稿，只追求写完

**写作顺序**：
1. [ ] 先写第3部分（实现步骤）- 最熟悉的部分
2. [ ] 再写第2部分（方案或原理）
3. [ ] 再写第4部分（示例/踩坑）
4. [ ] 再写第1部分（背景 & 问题）
5. [ ] 最后写第5部分（总结）

**初稿完成检查**：
- [ ] 5个部分都有内容
- [ ] 代码示例完整
- [ ] 逻辑基本通顺

---

## 第5步：优化

### 核心检查问题

**1. 内容连贯性**
- [ ] 是否跳过太多步骤？
- [ ] 每个步骤之间是否有过渡？
- [ ] 逻辑是否流畅？

**2. 前置知识**
- [ ] 是否假设了过多的前置知识？
- [ ] 必要的概念是否有说明？
- [ ] 专业术语是否有解释？

**3. 代码可靠性**
- [ ] 代码是否在 IDE 中验证过？
- [ ] 代码是否能运行？
- [ ] 代码是否有必要的注释？

**4. 可读性**
- [ ] 是否使用了视觉辅助（代码/图/列表）？
- [ ] 段落长度是否适中？
- [ ] 标题是否清晰？

### AI 辅助优化
- [ ] 使用 NotebookLM 等工具优化
- [ ] 获取反馈并迭代
