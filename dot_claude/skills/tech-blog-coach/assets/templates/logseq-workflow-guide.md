# Logseq 工作流：基于笔记修改文章初稿

## 使用场景

你的笔记和文章都存储在 Logseq 中（Markdown 格式），希望：
1. 基于已有的笔记内容
2. 完善和修改文章初稿
3. 确保文章符合费曼学习法的标准

---

## 使用方式

### 在调用技能时提供路径

```
笔记路径：/path/to/logseq-repo/pages/你的笔记.md
文章初稿路径：/path/to/logseq-repo/pages/你的文章初稿.md
```

### 示例提示

```
我正在写一篇技术文章，需要帮助。

笔记文件：/Users/xxx/logseq-repo/pages/Rust 所有权系统.md
文章初稿：/Users/xxx/logseq-repo/pages/使用 Rust 所有权处理内存安全.md

请基于我的笔记来帮助我完善这篇文章。
```

## 工作流程

### 第1步：读取和分析

**读取笔记文件**：
- 提取笔记中的核心概念
- 识别代码示例
- 收集参考资料和链接
- 理解笔记的组织结构

**读取文章初稿**：
- 分析当前的文章结构
- 识别已经完成的部分
- 找出薄弱或缺失的部分
- 评估是否符合5部分结构

**对比分析**：
- 笔记中有哪些内容文章中没有用到？
- 文章的哪些部分可以从笔记中获得补充？
- 当前文章的薄弱环节是什么？

---

### 第2步：基于笔记优化文章

#### 检查5部分结构

**1. 背景 & 问题**

**检查要点**：
- 是否有明确的问题场景？
- 是否说明了读者能学到什么？

**从笔记中提取**：
- 问题背景和动机
- 你最初学习时的困惑
- 为什么这个问题重要

**补充方式**：
```

→ 转化为文章开头：

本文将帮助 you 理解 Rust 的所有权系统，并学会如何在编译时保证内存安全。"
```

**2. 方案或原理**

**检查要点**：
- 核心概念是否有清晰解释？
- 是否使用了代码、图、列表辅助说明？

**从笔记中提取**：
- 概念定义和说明
- 官方文档的摘录
- 你对原理的理解

**补充方式**：
```

"- 所有权：每个值有一个所有者
- 移动：let b = a; a 被移动了"

→ 转化为文章：
"### 核心概念

**所有权规则**：
1. Rust 中的每个值都有一个所有者（owner）
2. 同一时间只能有一个所有者
3. 当所有者离开作用域时，值被丢弃

**移动语义**：
```rust

// println!("{}", a); // 错误！a 不再有效
```
"
```

**检查要点**：

**从笔记中提取**：

Rust 提供了多种创建字符串的方式：

```rust
// 方法1：从字面量创建

// 方法2：使用 to_string()

// 方法3：使用 String::new()

```

**关键说明**：
- `String::from()` 和 `to_string()` 效果相同
- `String::new()` 创建空字符串，需要后续添加内容

## Logseq 笔记的特点和转化

### Logseq 笔记的常见格式

**属性块**：
```markdown
---
type:: concept
tags:: #rust #ownership
status:: learning
---
```

**转化处理**：文章中通常不需要，保留核心信息即可

**内部链接**：
```markdown
这和 [[借用]] 有关
```

**转化处理**：
- 如果在 Logseq 内部发布：保留 `[[...]]` 格式
- 如果导出到其他平台：转换为普通文本或 Markdown 链接

**块引用**：
```markdown
这是我的笔记 ((block-id))
```

**转化处理**：展开被引用的内容，不要使用块引用

**折叠块**：
```markdown
   collapsed:: true
详细内容...
```

**转化处理**：展开所有内容，文章不应该有折叠

### 常见错误

**错误1：使用已移动的值**

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1;
    println!("{}", s1);  // 错误：use of moved value
}
```

**错误信息**：
```
error[E0382]: use of moved value: `s1`
 --> src/main.rs:4:20
  |
2 |     let s2 = s1;
  |              -- value moved here
3 |
4 |     println!("{}", s1);
  |                    ^^ value used here after move
```

**解决方案**：
1. 使用 `s1.clone()` 创建副本
2. 使用引用 `&s1`（借用）
3. 重新设计代码，让 s1 不再需要

**错误2：函数中的所有权转移**

```rust
fn main() {
    let s = String::from("hello");
    takes_ownership(s);  // s 的所有权被转移
    // println!("{}", s);  // 错误！s 不再有效
}

fn takes_ownership(some_string: String) {
    println!("{}", some_string);
} // some_string 离开作用域，内存被释放
```

**解决方案**：使用引用

```rust
fn main() {
    let s = String::from("hello");
    borrow(&s);  // 借用 s，所有权保留
    println!("{}", s);  // 正确！s 仍然有效
}

fn borrow(some_string: &String) {
    println!("{}", some_string);
} // some_string 离开作用域，但没有所有权，不会释放
```"

## 注意事项

### Logseq 特有格式

**保留的格式**：
- Markdown 标题（# ## ###）
- 代码块（```）
- 列表（- 或 1.）

**可能需要调整的格式**：
- Logseq 属性块（`---` 包围的部分）
- 内部链接 `[[...]]`（根据发布平台决定）
- 块引用 `((...))`
- 折叠块 `collapsed:: true`

### 文章 vs 笔记的区别

**记住**：
- 笔记是给自己看的 → 可以简略、可以跳跃
- 文章是给别人看的 → 必须完整、必须连贯
