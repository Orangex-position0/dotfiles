# 技术博客写作检查清单（基于费曼学习法）

## 核心检查问题（最重要）

发布前必须确保以下三个问题的答案都是"是"：

### 1. 内容连贯性
- [ ] **是否跳过太多步骤？**
  - 检查：每个步骤之间是否有清晰的逻辑连接
  - 检查：读者能否跟上思路而不需要"脑补"
  - 检查：是否有"显然"、"显而易见"等跳跃性表述

### 2. 前置知识
- [ ] **是否假设了过多的前置知识？**
  - 检查：专业术语是否有第一次出现时的解释
  - 检查：是否假设读者了解某些背景而没有说明
  - 检查：面向"比自己稍弱一级"的读者，内容是否合适

### 3. 代码可靠性
- [ ] **代码是否能运行？**
  - 检查：所有代码是否在 IDE 中验证过
  - 检查：代码片段是否完整（非截断）
  - 检查：依赖和版本是否有说明

---

## 结构完整性检查

基于5部分固定结构：

### 1. 背景 & 问题
- [ ] 介绍了文章的背景和定位
- [ ] 明确告诉读者能学到什么
- [ ] 有具体的问题场景引发共鸣
- [ ] 内容占比约 10-15%

### 2. 方案或原理
- [ ] 核心概念分点讲述
- [ ] 使用了代码/图/列表辅助说明
- [ ] 原理讲解清晰易懂
- [ ] 内容占比约 25-30%

### 3. 实现步骤
- [ ] 每个步骤有明确的目标
- [ ] 步骤之间有过渡
- [ ] 代码示例完整可运行
- [ ] 内容占比约 30-35%

### 4. 示例 / 踩坑
- [ ] 有完整的可运行示例
- [ ] 列出常见错误和解决方案
- [ ] 说明边界情况
- [ ] 提供调试技巧
- [ ] 内容占比约 15-20%

### 5. 总结
- [ ] 快速总结核心要点（3-5个）
- [ ] 明确适用场景
- [ ] 诚实说明局限性
- [ ] 提供延伸方向或资源
- [ ] 给予读者鼓励
- [ ] 内容占比约 5-10%

---

## 可读性检查

### 语言表达
- [ ] 使用简单的语言解释复杂概念（费曼学习法）
- [ ] 避免冗长的句子（每句不超过20字为佳）
- [ ] 段落长度适中（3-5句话）
- [ ] 避免过度使用专业术语

### 视觉辅助
- [ ] 使用列表呈现要点
- [ ] 使用代码块展示代码（带语法高亮）
- [ ] 关键内容有加粗或强调
- [ ] 复杂概念有类比或比喻

### 结构清晰
- [ ] 标题准确反映内容
- [ ] 章节之间过渡自然
- [ ] 重要信息不埋没在长段落中
- [ ] 有明确的逻辑主线

---

## 代码质量检查

### 完整性
- [ ] 代码片段完整，非截断版本
- [ ] 包含必要的导入和依赖
- [ ] 有配置或环境说明

### 可运行性
- [ ] 代码在 IDE 中验证过
- [ ] 提供了运行结果或输出
- [ ] 说明了运行环境（版本、平台）

### 可读性
- [ ] 代码块指定了语言类型
- [ ] 关键逻辑有注释
- [ ] 变量命名清晰
- [ ] 代码格式规范

---

## 技术准确性检查

- [ ] 技术信息准确无误
- [ ] 使用了当前的 API 和版本
- [ ] 引用了正确的文档
- [ ] 承认方法的局限性
- [ ] 没有夸大或误导性表述

---

## 发布准备检查

### 元信息
- [ ] 标题：包含核心主题，格式为"使用 xxx 处理 xxx"
- [ ] 摘要：一句话说明核心价值
- [ ] 标签：选择3-5个相关标签
- [ ] 分类：归入正确的分类

### 格式
- [ ] Markdown 语法正确
- [ ] 链接全部有效
- [ ] 图片加载正常（如有）
- [ ] 移动端显示良好

### SEO 优化（可选）
- [ ] 标题包含关键词
- [ ] 描述包含核心价值
- [ ] 图片有 alt 文字
- [ ] URL 简洁有意义

## 自我审查问题

### 内容层面
1. 文章是否实现了"能学到什么"中承诺的内容？
2. 目标读者（比自己稍弱一级）能否轻松理解？
3. 是否有必要的概念被遗漏？
4. 代码示例是否完整且可验证？

### 结构层面
1. 5个部分的比例是否合理？
2. 章节之间的过渡是否自然？
3. 是否有冗余或重复内容？
4. 标题和副标题是否准确反映内容？

### 表达层面
1. 是否用简单的语言解释了复杂概念？
2. 是否有足够的视觉辅助（代码/图/列表）？
3. 术语使用是否一致？
4. 语气是否适合目标读者？

---

## 常见问题检查表

### 内容问题
- ❌ 跳过中间步骤，逻辑跳跃
- ❌ 假设读者了解某些背景但没有说明
- ❌ 代码示例不完整或无法运行
- ❌ 没有解释代码的工作原理
- ❌ 忽略了错误处理和边界情况

### 结构问题
- ❌ 没有明确告诉读者能学到什么
- ❌ 5个部分的比例失衡
- ❌ 章节之间缺乏过渡
- ❌ 重要信息埋没在长段落中
- ❌ 总结只是重复前面的内容

### 可读性问题
- ❌ 句子过长，难以理解
- ❌ 段落过长，视觉疲劳
- ❌ 术语过多，缺乏解释
- ❌ 缺少示例和类比
- ❌ 没有使用视觉辅助元素

---

## 改进建议

### 如果发现"跳步骤"问题
- 补充中间的逻辑推导
- 添加过渡句说明
- 提供更多上下文信息

### 如果发现"假设过多前置知识"问题
- 在第一次出现术语时添加解释
- 添加"前置知识"部分
- 用类比帮助理解复杂概念
- 链接到相关的基础资料

### 如果发现"代码不可运行"问题
- 在 IDE 中完整运行一遍
- 补充缺失的导入和配置
- 说明依赖和版本要求
- 提供完整的示例代码
