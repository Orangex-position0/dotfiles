# PPP 工作文档生成器

## 技能概述

生成结构化的 PPP (Progress-Plans-Problems) 工作文档，帮助用户快速创建简洁、结果导向的工作汇报。

## 核心原则

- **结果导向**：将"做了什么"转换为"做成了什么"
- **简短精炼**：每个部分限制在 1-3 句话
- **结构清晰**：Progress → Plans → Problems 三段式结构

## 工作流程

### 第一步：引导用户输入

根据用户输入的详细程度，选择以下两种方式之一：

**方式 A：引导式提问（推荐）**
当用户提供的信息较分散或未明确结构时，逐一询问：

```
请按以下结构提供信息：

👤 姓名：
（你的名字或昵称）

✅ Progress（进展）：
今天完成的工作和成果

📋 Plans（计划）：
下一步要做什么（短期可执行的）

⚠️ Problems（问题/风险）：
当前遇到的阻碍或风险
```

**方式 B：自由输入 + AI 提取**
当用户已提供完整段落时，直接从中提取结构化信息，无需二次询问。

### 第二步：内容优化

对用户输入进行以下优化：

1. **转换为结果导向表达**
   - 识别过程描述（"做了"、"花了时间"）并改为结果描述（"完成"、"实现"、"修复"）
   - 突出成果和价值（"提升了X%"、"修复了Y bug"、"交付了Z功能"）

2. **精简句子**
   - 删除冗余词汇（"我觉得"、"可能"、"打算"）
   - 合并重复描述
   - 确保每个部分 1-3 句话

3. **保留关键信息**
   - 具体指标（数据、百分比）
   - 重要风险点
   - 可执行的下一步计划

### 第三步：生成 Markdown 文档

使用以下格式输出（使用 YAML Frontmatter + Emoji）：

```markdown
---
date: YYYY-MM-DD
author: [姓名]
week: [可选：周次，如 2026-W13]
---

# 工作汇报 - PPP

## ✅ Progress（进展）
- [完成的具体成果1]
- [完成的具体成果2]

## 📋 Plans（计划）
- [下一步计划1]
- [下一步计划2]

## ⚠️ Problems（问题/风险）
- [当前问题或风险1]
- [当前问题或风险2]
```

**元数据字段说明**：
- `date`：汇报日期（必填）
- `author`：汇报人姓名（必填）
- `week`：ISO 周次（可选，格式：YYYY-Www）
- 可根据需要添加其他字段（如 `sprint`、`project`、`tags` 等）

## 输出示例

**用户输入**：
```
姓名：张三

我今天主要在调试用户登录的问题，花了挺长时间，最后发现是token过期没处理好的问题，修复后应该就好了。然后还开了个会讨论新功能的需求。
接下来我想把测试写完，然后看看能不能优化一下数据库查询。
没什么大问题，就是第三方API有时候挺慢的。
```

**优化后输出**：
```markdown
---
date: 2025-03-28
author: 张三
week: 2025-W13
---

# 工作汇报 - PPP

## ✅ Progress（进展）
- 修复了用户登录token过期的处理bug
- 完成新功能需求讨论会议

## 📋 Plans（计划）
- 完成登录功能的测试用例
- 优化数据库查询性能

## ⚠️ Problems（问题/风险）
- 第三方API响应不稳定，需要监控和优化
```

## 质量检查清单

生成文档后，确保满足：
- [ ] 每个部分 1-3 句话
- [ ] Progress 使用"完成/实现/修复"等结果动词
- [ ] Plans 是可执行的短期计划
- [ ] Problems 描述了具体阻碍或风险
- [ ] 整体简洁明了，无冗余信息
